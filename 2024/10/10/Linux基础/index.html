Linux 学习笔记 - By yexiaonan

Linux是一个开源的操作系统 不要尝试去死记硬背一个命令 要养成习惯来使用命令

Linux里面Linux内核处理任何东西都要有一个 数字节点

放在前面：

Root用户可执行的命令放在：

/sbin

/usr/sbin

(bin-binary二进制文件)

所有用户可执行的命令放在：

/bin

/usr/bin

【一】Linux命令的结构：

命令(Command)+选项(Options)+参数(Argument)

命令：通常情况下指的是某个功能

选项：通常来讲是对命令的修饰，对命令功能的补充

参数：通常来讲是想要操作的对象

参数可以省略，但是不代表他没有

Linux语法：

命令和选项之间使用空格来进行分隔

命令和参数之间使用空格来进行分隔

选项和选项之间使用空格来进行分隔

选项和参数之间使用空格来进行分隔

参数和参数之间使用空格来进行分隔

多个命令使用 “；” 进行分隔

选项和参数可以省略；但是命令不行

【二】文件处理命令：

Linux基础命令

（1）ls [选项] [文件或目录]

  • 显示当前所处目录下的所有文件（隐藏文件除外）

执行权限：所有User

-a 显示所有文件，包括隐藏文件

-l 显示文件的详细信息

-d 查看目录的属性

（2）pwd [路径]

  • 显示当前所在的工作目录

（3）cd [路径]

  • 切换目录

cd .. 回到上一级目录

（4）touch [路径] [文件名]

  • 创建一个 文件

[路径] 选项为可加可不加，如果不加路径，则在当前所处的目录下进行创建

（5）mkdir [目录名]

  • 在当前所处的目录下创建一个目录

（6）cp [源文件目录/文件] [复制到的目录/文件]

  • 复制一个目录或者文件到指定位置

（如果要复制的是一个目录，那么就要在选项里面加上 “-R” ，例如 cp -R /test /root）

（7）mv [源文件或目录] [目的目录]

  • 移动一个文件，改名

（8）rm [文件或目录]

  • 删除文件，目录

如果需要删除目录要在选项处加上“-r”，如果单单删除文件则不需要加上“-r”

（选项加上“-f”可以强制删除，而不在询问是否删除）

（9）cat [文件名]

  • 查看文件的内容（适用于较短的文件显示，太长的显示不完全）

（10）more [文件名]

  • 分页显示文件的内容（适用于较长的文件显示）

（空格）或f 显示下一页

（Enter） 显示下一行

q或Q 退出

（11）head -num [文件名]

  • 可以选择到 ”行“ 查看文件的内容

（不加选项默认显示前十行，如果想显示到几行，要在参数选项上加上 ‘-num’ 想要显示到多少行）

（12）tail -num [文件名]

  • 查看文件的最后几行

（13）ln - s [源文件] [目标文件]

  • 创建链接文件

-s：建立软链接文件（）。 （’-f‘选项可以强制添加链接，并且删除已经存在的链接）

如果不加 “-s” 选项，则建立硬链接文件；

（软连接类似为Windows的快捷方式）

（硬连接类似为Windows的拷贝，它是创建了一个与源文件大小一样的文件；他与拷贝不一样的是他会同步更新；拷贝过的文件会重新更替，时间值，如果不希望改变时间的话需要再选项处加上“-p”）

硬链接不能跨越文件系统；软链接则没有这个限制

（5）any command [选项]

输入/输出重定向

同标准 I/O 一样，Shell对于每一个进程预先定义3个文件描述字（0，1，2）.分别对应于：

0 （STDIN）标准输入；

1 （STDOUT）标准输出；

2 （STDERR）标准错误输出。

选项：

’>‘输出重定向；（把一个文件写入到指定文件里）

  • {ls -l /tmp > /tmp.msg
  • date >> /tmp.msg}

两个’>>’可以追加内容写入到文件而不覆盖掉之前的内容

‘<’输入重定向；（把一个文件里面的内容以wall的形式显示出来）

  • {wall < /etc/motd}

‘2>’错误输出重定向（如果执行Command的时候出错，可以把Error以文件的形式记录下来） 

  • {cp -R /usr /backup/usr.bak 2> /bak.error}

【八.二】Linux用户分为三种：

*超级用户（root，UID=0）

*普通用户（UID=500-60000）

*伪用户（UID=1-499）

每一个用户都至少属于一个用户组

每个用户组可以包括多个用户

同一个用户组的用户享有该组共有的权限

*mail,news,games,apache,ftp,mysql及sshd等，与Linux系统的进程相关

2.伪用户通常不需要或无法登录系统

3.可以没有宿主目录
